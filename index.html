snoose

by: Julia Hicks

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because I was shown a drawing program and drew a random girl. I then decided to use it it my game to make something similar to hangman.

2. What I Learned

My teacher made a suggestion to, on the end screen (after you've finished playing), to add photos of the word the player was guessing. It was really fun to add and made the game look a little more interesting. Also, another teacher reminded me to make sure that the textbox with the player's guesses clears after each round, because for some reason I forgot to until the very end.

3. Architecture of My Game

I have three main groups of classes. The Snoose class itself, the screen classes, and the game panel. The Snoose class creates the frame that the rest of the game runs on and sets its size. The gamePanel class controls changing through the different gameplay panels and creates the timer that times the game. The menuState, gameState, and endState classes build the three pain parts of the game (the menu, the game itself, and the end screen). Lastly, the screen class creates the words that the player will be guessing and holds all the hints for every word. Also, it holds the images for each word on the endScreen.

4. Technical Implementation

Even though it's not really coding related, I like the photos on the endScreen. It was really fun to have the ability to choose any photo I want and place it anywhere I wanted.

5. Most Challenging Part

Honestly, the most difficult part of this game was how long it took. I am a very slow worker and on top of that, lots of stuff went wrong. But now, it's finished :)

6. Future Fixes and Updates

One big thing that Snoose could use is more detailed graphics because the game looks a little bland at the moment.
